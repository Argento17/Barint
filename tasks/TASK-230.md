# TASK-230 — Salty-snacks v4 copy remediation

Owner reviewed the shipped salty-snacks v4 shelf and found the auto-generated copy is "really
off." Confirmed defects in `bari-web/src/data/comparisons/salty_snacks_frontend_v4.json`
(copy is generated by `03_operations/bsip0/scrape/salty_snacks_real/03_build_frontend_v4.py`):

## Confirmed copy defects (with evidence)

1. **FRAMEWORK LEAKAGE — critical.** `limitingFactors` renders `"מוצר אולטרה-מעובד (NOVA 4)"`
   verbatim to consumers. "NOVA" is a banned framework term (Content Hard Rule 1, QA leakage
   checklist). It even appears on a product whose `novaGroup` is **3** (פריכיות תירס עם סויה
   וזרעי פשתן) — leakage AND a contradiction. Remove all NOVA / framework vocabulary from every
   consumer string; express processing in plain consumer language ("חטיף מעובד מאוד" only if accurate).
2. **RECOMMENDATION LANGUAGE — banned.** `bottomLine` says `"...עדיף לבחור חטיף עם פרופיל נקי יותר"`
   on 13 products. Bari describes, never prescribes (no מומלץ/כדאי/עדיף לבחור). Rewrite to a
   descriptive close.
3. **CONFUSING LABEL.** Confidence label `"מבוסס על נתוני תזונה בלבד"` (34/47 products) reads as
   jargon — owner asked "what does לוח תזונה even mean?". Reword so a shopper understands it means
   "the ingredient list wasn't available, so the score uses the nutrition panel only" — in plain Hebrew.
4. **DESCRIPTION QUALITY.** insightLine/bottomLine are formulaic and weak (e.g. "חטיף ממוצע למדף —
   לא בולט לטובה ולא נופל בבירור"). Raise to the editorial-intelligence v3 / assertive-writing bar:
   finding-first, product-specific, no template repetition, no score-mechanism explanation.

## Scope

- Audit ALL consumer strings in v4 (insightLine, expansion.bottomLine, positiveSignals,
  limitingFactors, confidenceLabel/confidence_label_he, comparisonContext) for leakage +
  recommendations + quality. Run each through the offline `hebrew_readability.is_clean` gate.
- Produce the corrected **copy generation rules + replacement templates** (by product archetype:
  high-fiber/protein baked snack, plain rice cake/popcorn, extruded puff, sweet-leaning snack,
  panel-only/partial-data product). This is what the Data Agent's build script will implement
  (TASK-231). Do not hand-edit 47 strings if a template fix is cleaner — but provide exact Hebrew.
- Note: some products have bad DATA (garbled/English ingredients, missing sodium) being fixed in
  TASK-231 (Data). Write copy rules that degrade gracefully when a field is missing; flag any
  copy you cannot write until the data is fixed.

## Acceptance criteria
- [ ] Zero framework terms (NOVA/BSIP/cap/floor/dimension…) in any consumer string — `is_clean` passes
- [ ] Zero recommendation language (מומלץ/כדאי/עדיף לבחור…) — descriptive closes only
- [ ] Confidence label reworded to plain, shopper-legible Hebrew
- [ ] Corrected copy rules/templates delivered for TASK-231 to implement; sample rewrites for ≥6 archetypes
- [ ] Max one em-dash per paragraph rule honored
